Coin Construction: Who gets paid, by whom and how

  • Administrative

    Practical reality of the number or value of transactions in the current block should indicate coin strength and be some multiplier of budget. 1/active-forging-nodes * Forging Fee

  • Investors

    Investors are Financial Service Consumers looking for return on investment, not specifically looking to use the product. Placing arbitrary value on coin is not decentralized. Not sure of good model.

  • Developers

    Investor Buyin Distributed 100% to 3 rounds of development for each accepted team based on feeback, criticism, encouragement of all involved developers. For instance, if a team of 5 wins the right to produce API code for Android, then after the first round ALL of the other developers from all the other teams have the right to vote on quality, perspective, ability to meet goals, etc of that team (and all the other teams). A team with non-passing votes will not get to continue to next round. Reorganize, and resubmit. Highest voted projects go to round 2, etc. Developers can take Nominated investment currency, or speculatively take {some equivalent} of Big Market Coin.

  • Miners / Forgers

    Typical: Reserved currency rewards eventually transitions to nodes setting forging fees. Reserved currency rewards are equally distributed as User rewards. Fees are not. Maybe look at RaiBlocks and Iota's ideas as well.

  • Users

    Real world adoption is reflected by real world transactions. The higher the {transactions / node} ratio, the more "alive" the network is. Set up Network Marketing style 7 tier rewards where each layer up gets 1/layer reward. Consider app download and installation as a transaction which rewards the person that downloads the app, but becomes "downstream" to person that first transacts. Make it so that users encourage and actively build commerce communities. Phase out adoption incentive with reserved currency for miners.
